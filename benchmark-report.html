4 Token Cost Growth Over Conversation

Selected secrets: Manual vs Eager vs Lazy (EN, turns 5–50)

📈 The Leak Tax Effect

Every time a secret appears in conversation history, it costs tokens on every subsequent API call. A 3243-char RSA key adds ~600 tokens to history permanently — multiplying cost over time.

📉 Zocket's Flat Overhead

Zocket's fixed overhead (tool schemas + system prompt) is constant per call. The secret value never appears in context — only the key name does. This is why Zocket wins for long conversations.

⚡ Lazy Mode Advantage

Lazy mode registers only 2 meta-tools initially, saving 119 tokens per call vs eager. Over 50 calls that's 5950 tokens saved — roughly equivalent to 8 JWT tokens.

9 Scenario: Auto-Compacting (Context Compression)

🗜 What compaction means for Manual

When Claude Code compacts conversation history, secrets shared manually are gone or unreliably preserved. The user must re-paste the secret. Leak tax restarts. Each compaction event = full cost re-incurred.

🔒 What compaction means for Zocket

The vault is external state — completely unaffected by context compression. After any compaction, the AI just calls the tool again. Zero re-injection. Works after 1 or 1000 compactions.

📊 Model: 50 turns → compact → 50 turns

History compressed to 15% after phase 1. Phase 2 Manual re-injects the secret at the first use. Phase 2 Zocket behaves identically to phase 1 — no state change needed.

Total tokens: Phase 1 + Phase 2 (after compaction). 100 turns total.

Phase 1 = pre-compaction (50 turns). Phase 2 = post-compaction (50 more turns). Manual secret re-injected at first use after compaction.

10 Scenario: Mid-Session Secret Injection

⏱ Manual: position-dependent cost

Injecting a secret at turn 50 of 100 is cheaper than turn 1 — but only because the secret pollutes fewer remaining turns. You pay with workflow constraints: you can't use the secret early without cost.

🎯 Zocket: position-independent

Tool call cost is identical at any turn. Whether you access the secret at turn 1 or turn 99, the token cost is the same. This is "any-time access" — use secrets whenever needed, without planning around leak tax.

📈 100-turn session, injected at turns 1, 10, 25, 50, 75

17 Security Mode — Token Overhead Analysis

🔁 Confirmation round-trip cost

When a command triggers MEDIUM risk (e.g. SUSPICIOUS_DOMAIN), MCP returns requires_confirmation instead of executing. The AI reads it, sends a second call with confirm:true. Total overhead per use: 123 tokens (47 response + 26 AI confirm + 50 retry call).

📊 Realistic vs worst-case

Sec-20%: 1 in 5 secret uses triggers confirmation (realistic for SUSPICIOUS_DOMAIN when not all keys go to flagged APIs). Overhead over 50 messages: +246 tok vs Eager.
Sec-100%: every use confirms (adversarial scenario). Overhead over 50 messages: +1230 tok vs Eager.

✅ Security is affordable

For long secrets (JWT, SSH RSA-4096), security overhead is <5% of Manual cost — negligible. For short secrets (passwords, API keys), security adds relatively more overhead but these are already cheaper with Manual — use security for protection, not token savings.
